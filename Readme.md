# eufymake2mqtt

A Java-based bridge service that mirrors data from the **eufyMake cloud MQTT broker** to a **local MQTT broker**. This
allows you to integrate eufyMake device data into your own smart home setup (e.g., Home Assistant,
Node-RED, [STEM](https://github.com/LinzN/Stem), or any other MQTT-based system) without relying on the cloud.
 
---

## How It Works

```
eufyMake Cloud MQTT  ‚îÄ‚îÄ‚ñ∫  eufymake2mqtt  ‚îÄ‚îÄ‚ñ∫  Local MQTT Broker
```

The service connects to the eufyMake cloud MQTT broker, subscribes to device topics, and re-publishes all received
messages to your local MQTT broker. This gives you full local access to your eufyMake device data.
 
---

## Requirements

- Java 11 or higher
- Maven 3.x
- A running local MQTT broker (e.g., [Mosquitto](https://mosquitto.org/))
- `default.json` generated by [ankermake-m5-protocol](https://github.com/Django1982/ankermake-m5-protocol) (see below)

---

## Setup

### Step 1 ‚Äî Generate `default.json` with ankermake-m5-protocol

The `eufy/default.json` or maybe also called `login.json` file contains your eufyMake account and printer credentials.
It is **not included** in this repository and must be generated using
the [ankermake-m5-protocol](https://github.com/Django1982/ankermake-m5-protocol) tool. Please refer to that project for
instructions.

Once generated, copy the file into the `eufy/` folder of this project:

```bash
cp /path/to/default.json /path/to/eufymake2mqtt/eufy/default.json
```

 
---

### Step 2 ‚Äî Build eufymake2mqtt

```bash
git clone https://github.com/LinzN/eufymake2mqtt.git
cd eufymake2mqtt
mvn clean package
```

The compiled JAR will be placed in the `target/` directory.
 
---

### Step 3 ‚Äî First Run & Configuration

On the first run, a `config.yml` is automatically created in the project root directory. Stop the service, open
`config.yml`, and fill in your local MQTT broker settings:

```bash
java -jar target/eufymake2mqtt.jar
# Stop after config.yml is generated, then edit it:
nano config.yml
```

| Parameter       | Description                                      |
|-----------------|--------------------------------------------------|
| `mqtt.host`     | Host of your local MQTT broker                   |
| `mqtt.port`     | Port of your local MQTT broker (default: `1883`) |
| `mqtt.username` | Username for your local broker (if any)          |
| `mqtt.password` | Password for your local broker (if any)          |

 
---

### Step 4 ‚Äî Run

```bash
java -jar target/eufymake2mqtt.jar
```
